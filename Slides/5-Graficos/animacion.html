Para definir una animación primero debemos definirla mediante la etiqueta @keyframes y despues asignarsela al elemento deseado.

Mejorar el rendimiento de las animaciones y transiciones.

Para mejorar el rendimiento de animaciones complejas debemos tener en cuenta que las animaciones en 2D son calculadas y ejecutadas desde la CPU, mientras que las animaciones 3D son ejecutadas desde la GPU. Por ello, a veces es mejor llevarnos la animación al espacio tridimensional para mejorar su rendimiento. Por ejemplo, añadiendo la propiedad transform: translateZ(0);

@keyframes

@keyframes name {
    from {
        opacity: 0;
    }

    to {
        opacity: 1;
    }
}
@keyframes name {
    from {
        opacity: 0;
    }
         
    25% {
        opacity: 1;
    }

    75% {
        opacity: .5;
    }
    
    to {
        opacity: 1;
    }
}
@keyframes name {
    from {
        opacity: 0;
    }

    25%, 50% {
        opacity: .5;
    }

    to {
        opacity: 1;
    }
}

animation-name y animation-duration

.anime-name {
    animation-name: rotateAnimation;
    animation-duration: 10s;
}

animation-timing

.anime {
    animation-name: changeOpacity;
    animation-duration:  1s;
    animation-timing-function:  ease-in-out;
}

animation-delay

.anime-timing {
    animation-name: changeOpacity;
    animation-duration:  1s;
    animation-timing-function:  ease-in-out;
    animation-delay:  5s;
}

animation-iteration-count

Podemos indicar el número de veces que queremos que la animación se repita, pero también podemos decir que la animación se repita siembre mediante la palabra infinite.

.anime-iteration {
    animation-name: rotateAnimation;
    animation-duration: 5s;
    animation-iteration-count: infinite;
}

animation-direction

Mediante esta propiedad podemos indicar en que dirección queremos que la animación se ejecute. Si desde el fotograma inicial hasta el final o a la inversa. Los valores que puede tomar son:

  • normal: Se ejecuta desde el principio hasta el final.
  • reverse: Se ejecuta desde el fotograma final hasta el inicial, es decir, marcha atrás.
  • alternate: Va alternando cada vez que se ejecuta una iteración, primero hacia adelante y despues hacia atrás.
.anime-direction {
    animation-name: rotateAnimation;
    animation-duration: 5s;
    animation-iteration-count: infinite;
    animation-direction: reverse;
}

animation-fill-mode

.anime-mode {
    animation-name: changeBgColor;
    animation-duration: 4s;
    animation-iteration-count: 3;
    animation-fill-mode: fordwards; 
}

animation-play-state

.anime-pause {
    animation-name: rotateAnimation;
    animation-duration: 5s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
}

.anime-pause:hover {
    animation-play-state: paused;
}

Shorthand

Podemos establecer todas las propiedades a la vez mientras las indiquemos en el siguiente orden: animation-name, animation-duration, animation-timing-function, animation-delay animation-iteration-coun, animation-direction y animation-fill-mode.

.anime {
    animation: rotateAnimation 2s ease-in-out 5s infinite alternate-reverse forwards;
}
