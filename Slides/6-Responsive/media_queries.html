Media Queries

  • Diseño responsive
  • Introducción a RWD

Las media queries nos permiten aplicar unos estilos en función del medio en el que estamos, normalmente la resolución del dispositivo. Los cambios más habituales que solemos aplicar con media queries son:

  • Ajustar el tamaño de las columnas del layout.
  • Flexibilidad de los anchos de los elementos.
  • Adaptar los espacios entre elementos. Por ejemplo: espacio entre cabeceras, etc.
  • Adaptar el tamaño de la fuente.
  • Cambiar los menus de navegación.
  • Esconder contenido para resoluciones móviles.
  • Modificar las imágenes.

Las media queries se pueden aplicar a los siguientes medios:

  • all: Se aplicaría a todos los medios.
  • print: Se aplicaría solo cuando se imprime la página.
  • screen: Para todas las pantallas de ordenador, tablet, móvil, etc.
  • speech: Utilizada para screenreaders.

Tenemos más medios disponibles sin embargo son los menos utilizados, como: braille, embossed, handheld, projection, tty y tv.


Además podemos aplicar la media query según unas características determinadas:

  • width, min-width y max-width: Indica el ancho del documento que se está visualizando.
  • height, min-height y max-height: Indica el alto del documento.
  • resolution: Indica la resolución del dispositivo, medida en dpi o dpcm.
  • orientation: Según la orientación del dispositivo, pudiendo ser landscape o portrait.
  • aspect-ratio: Indica la relación de aspecto del dispositivo, por ejemplo 16/9.

También hay otras propiedades menos utilizadas como: color, color-index, device-aspect-ratio, device-width, device-height, grid, monochrome o scan.

Como incluir media queries

Podemos incluir las media queries en nuestro proyecto de diferentes modos, mediante un llamada a un fichero independiente, incluyendolas junto a las hojas de estilos o referenciandolas desde una hoja de estilos independiente.

Links

La ventaja de este método es que solo se cargarán los estilos necesarios para la resolución en la que nos encontramos. El problema es que implica una llamada extra al servidor.

<link href="css/small.css" rel="stylesheet" media="(width: 480px)">

@import

Mediante esta etiqueta estamos importando nuestras media queries de un fichero externo a la hoja de estilos desde la que se llama. La ventaja de este método es que tenemos una hoja de estilos independiente donde almacenar nuestras media queries.

@import url(css/small.css) (max-width: 320px);

@import url(css/small.css) (max-width: 320px) and (max-width: 768px);

@media

Mediante la etiqueta @media estamos generando un bloque donde indicaremos los estilos que queremos aplicar segun el parametro de la media query que indiquemos. Independientemente de que guardemos nuestras media queries en un fichero independiente o no, siempre debemos declararlas dentro de la etiqueta @media.

@media (min-width: 768px) and (max-width: 1024px) {
    .media {
        width: 50%;
        background: #333;
    }
}

@media (min-width: 1200px) {
    .media {                                 
        width: 75%;
        background: #29A9DF;
    }
}

@media (min-width: 1500px) {
    .media {
        width: 100%;
        background: #0170BA;
    }
}
